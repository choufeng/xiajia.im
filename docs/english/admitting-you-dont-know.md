# Admitting You Don't Know

> 日期：2026-06-21 · 场景：体面承认「我不知道」

## 🎯 为什么要练"大方承认不知道"

会上被问住，最糟的反应是硬编一个答案——日后翻车，信用破产。第二糟是支支吾吾。英语会议里「不知道」有专门的体面话术：诚实承认 + 承诺跟进 + 找对人。

五种策略覆盖所有「答不上来」的时刻：

| 策略 | 一句话用途 |
| --- | --- |
| Direct admit | 直接承认，不绕弯 |
| Admit + commit | 承认 + 承诺查 |
| Defer to expert | 转给更懂的人 |
| "Let me check" | 表示要核实 |
| Scope the unknown | 圈定不确定范围 |

记住顺序：**承认 → 承诺 → 转专家 → 要核实 → 圈范围**。诚实比装懂值钱十倍——尤其对工程师。

## 💡 五种体面承认策略

### 1. Direct admit — 直接承认不绕弯

**何时用**：被问到确实不知道的。别编、别拖，直接说不知道——比绕弯子专业。

**句式模板**：
- ***Off the top**, I'm not entirely sure.*
- ***To be honest**, I don't have that number offhand.*
- *I don't know the answer to that — let me find out.*

> **例**：Off the top of my head, I'm not entirely sure. To be honest, I haven't benchmarked that path yet.

### 2. Admit + commit — 承认 + 承诺跟进

**何时用**：光说「不知道」会显得敷衍。加一句「我去查」——把不知道变成可追踪的行动。

**句式模板**：
- *I don't have it now, but I can **verify** and share.*
- *Let me **look into** it and **get back to** you.*
- *Not sure offhand — I'll confirm by [时间].*

> **例**：I'd rather verify it against real numbers before committing.

### 3. Defer to expert — 转给更懂的人

**何时用**：你不知道，但知道谁知道。大方转介——既解决问题，又显得尊重同事专长。

**句式模板**：
- *[名字] would know this offhand.*
- *That's really [名字]'s area — let me ask them.*
- *[名字] runs [相关领域] — they'd have the data.*

> **例**：Lena runs the perf lab — she'd have the latest data. I'll look into it with her.

### 4. "Let me check" — 表示要核实

**何时用**：你有大致印象，但不敢打包票。明示「要核实」——给自己留余地，也显严谨。

**句式模板**：
- *I can give a guess, but I'd rather **verify**.*
- *Let me **check** before I commit to a number.*
- *I'd want to **look into** that before answering.*

> **例**：I can give you a guess, but I'd rather verify it against real numbers before committing.

### 5. Scope the unknown — 圈定不确定范围

**何时用**：你部分知道、部分不知道。圈出「知道的部分」和「不知道的部分」——比笼统说不知道专业得多。

**句式模板**：
- *I know [知道的部分], but [不知道的部分] I'd need to check.*
- *What I can speak to is [知道的部分]. The rest, I'll confirm.*
- *I'm confident on [知道的部分]; less sure on [不确定的部分].*

> **例**：I can speak to the general design, but the exact latency numbers I'd need to verify.
