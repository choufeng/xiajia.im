# Keeping Discussion on Track

> 日期：2026-06-21 · 场景：控场拉回主题、防止跑题

## 🎯 为什么要练"拉回正题"

线上会议最容易跑题——一句「这让我想起去年那个 bug」就能让 15 分钟的会聊偏 20 分钟。主持人最值钱的能力之一，是把跑偏的讨论礼貌地拽回来。

五种策略覆盖所有「该控场」的时刻：

| 策略 | 一句话用途 |
| --- | --- |
| Acknowledge + park | 认可后搁置，不伤人 |
| Time check | 时间提醒，逼收敛 |
| Refocus on agenda | 拉回议程项 |
| Table for later | 延后讨论，留记录 |
| Redirect question | 重定向到该讨论的人 |

记住顺序：**认可 → 时间 → 议程 → 延后 → 重定向**。拉回的核心是「先肯定对方没说错，再把它收起来」。

## 💡 五种拉回正题策略

### 1. Acknowledge + park — 认可后搁置

**何时用**：有人提出有价值但跑题的点。先认可它有价值，再收起来——既不否定对方，又不让会议跑偏。

**句式模板**：
- *Good point — let's **table that** for now.*
- *Worth discussing, but let's **parking lot** it.*
- *I hear you — let's capture that and move on.*

> **例**：Hold on, that's a bit of a tangent. Let's table that for now.

### 2. Time check — 时间提醒逼收敛

**何时用**：讨论开始发散，但没人意识到。报个时间——大家自然会收。

**句式模板**：
- *Quick time check — we've got [X] minutes left.*
- *We're over on this item — let's wrap it.*
- *Clock check: five minutes, two decisions left.*

> **例**：Quick time check — we've got eight minutes left.

### 3. Refocus on agenda — 拉回议程项

**何时用**：彻底跑偏时，直接点名议程。议程是你的挡箭牌——不是你在打断，是议程在推进。

**句式模板**：
- *Back to the agenda — we're on [议程项].*
- *The **scope** here is just [范围].*
- *Let's refocus on [议程项] — that's today's goal.*

> **例**：Back to the agenda — we're on the rollout plan. The scope here is just phase one.

### 4. Table for later — 延后讨论留记录

**何时用**：跑题的点值得深聊，但不是现在。承诺延后 + 记录——让对方放心它不会被丢。

**句式模板**：
- *Let's **parking lot** that and **circle back** next week.*
- *I'll note it — we can pick it up in the next sync.*
- *Let's defer that to a dedicated session.*

> **例**：Let's parking lot the caching debate and circle back to it next week.

### 5. Redirect question — 重定向到该讨论的人

**何时用**：某人在不属于他的话题上纠缠。把问题转给真正该回答的人——既终止跑偏，又推进议程。

**句式模板**：
- *That's really [名字]'s call — let's ask them.*
- *[名字], can you take that one?*
- *Let's defer to [名字] on that — back to [议程项].*

> **例**：That's really the platform team's call — let's refocus on our decision.
