# Buying Time

> 日期：2026-06-21 · 场景：会议中争取思考时间

## 🎯 为什么要练"拖时间"

会上突然被点名，脑子一片空白——沉默最致命。英语会议有一整套「合法拖延」的套话，让你边拖边想，还不显慌。

五种策略覆盖所有「需要几秒钟」的时刻：

| 策略 | 一句话用途 |
| --- | --- |
| Echo question | 复述问题，白赚 5 秒 |
| Signal thinking | 明示"我在想"，堵住催促 |
| Give partial | 先给部分答案稳住场面 |
| Ballpark | 给粗估，留余地 |
| Promise follow-up | 承诺会后给准确答复 |

记住顺序：**复述 → 示意思考 → 给部分 → 给粗估 → 承诺跟进**。目标不是答完美，是「不冷场、不乱答」。

## 💡 五种争取时间策略

### 1. Echo question — 复述问题白赚时间

**何时用**：问题抛来，大脑还没启动。把问题复述一遍——既确认理解，又白赚 5 秒思考。

**句式模板**：
- *So you're asking [复述问题]?*
- *Just to make sure I understand — [复述]?*
- *The question is [复述], right?*

> **例**：So you're asking how many concurrent users the gateway handles before it fails?

### 2. Signal thinking — 明示你在思考

**何时用**：复述完还要想。用一句话明示"我在想"，堵住对方的催促——沉默会被当不会，明示思考是专业的。

**句式模板**：
- ***Let me think** for a second...*
- *Good question. Let me work through this.*
- *Give me a moment to recall.*

> **例**：Good question. Let me think for a second...

### 3. Give partial — 先给部分答案稳场面

**何时用**：完整答案想不出来，但有一部分。先抛出来稳住场面，比干等强。

**句式模板**：
- *I can speak to [部分答案]. The rest I'd need to check.*
- *Part of the answer is [部分答案].*
- *What I do know is [部分答案].*

> **例**：Off the cuff, my first instinct says around two thousand.

### 4. Ballpark — 给粗估留余地

**何时用**：精确数字拿不准。给个 ballpark（粗估），并明示是估计——既答了，又给自己留了余地。

**句式模板**：
- ***Ballpark**, I'd say [粗估].*
- ***Roughly speaking**, around [粗估].*
- *Off the top, maybe [粗估] — but don't quote me yet.*

> **例**：Roughly speaking, yes — that was the configuration from last quarter's run.

### 5. Promise follow-up — 承诺会后给准确答复

**何时用**：当场给不了准确答案。大方承诺会后跟进——比硬编一个错的数字专业一百倍。

**句式模板**：
- *Let me verify and get back to you by [时间].*
- *I'll pull the data and share it after this.*
- *Can I confirm offline and circle back?*

> **例**：I'll pull the latest report and give you a solid figure by end of day.

## 💬 Dialogue

**A**: Quick one — how many concurrent users can the current API gateway handle before it falls over?
<span style="font-size:0.85em;color:var(--vp-c-text-2)">A：问个快的——现在的 API 网关能扛多少并发用户才会崩？</span>

**B**: Good question. **Let me think** for a second...
<span style="font-size:0.85em;color:var(--vp-c-text-2)">B：好问题。让我想一下……</span>

**A**: Take your time. Rough number is fine.
<span style="font-size:0.85em;color:var(--vp-c-text-2)">A：慢慢来。大概数字就行。</span>

**B**: **Off the cuff**, my **first instinct** says around two thousand, but I'd want to verify against the last load test.
<span style="font-size:0.85em;color:var(--vp-c-text-2)">B：凭印象，第一感觉大概两千，但我想跟上次压测对一下。</span>

**A**: Two thousand is a useful **ballpark**. Is that with the new rate limiter enabled?
<span style="font-size:0.85em;color:var(--vp-c-text-2)">A：两千是有用的粗估。那是开了新限流器的情况吗？</span>

**B**: **Roughly speaking**, yes — that was the configuration from last quarter's run. Things may have shifted.
<span style="font-size:0.85em;color:var(--vp-c-text-2)">B：粗略说是的——那是上季度那次的配置。可能变了。</span>

**A**: Makes sense. Can you confirm the exact number after this meeting?
<span style="font-size:0.85em;color:var(--vp-c-text-2)">A：合理。会后能确认准确数字吗？</span>

**B**: Sure — I'll pull the latest report and give you a solid figure by end of day.
<span style="font-size:0.85em;color:var(--vp-c-text-2)">B：行——我拉最新报告，今天下班前给你准确数。</span>
