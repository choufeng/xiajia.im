# Online Meeting Follow-ups

> 日期：2026-06-21 · 场景：线上会议接话技巧

## 🎯 为什么要练"接话"

线上会议里，同事讲完后那一秒钟最难熬——你知道该开口，但脑子还在翻译上一句。

这一篇不背单词，背**句式**。下面五种接话策略覆盖了会议中 90% 的「该我说话了」时刻：

| 策略 | 一句话用途 |
| --- | --- |
| Paraphrase | 复述确认，争取思考时间 |
| Build on | 补充构建，展示在听 |
| Clarify | 提问澄清，争取发言权 |
| Pivot | 承接转向，引到你关心的话题 |
| Hand off | 总结交接，体面收尾 |

记住顺序：**听懂 → 补充 → 追问 → 转向 → 收尾**。任意一步都能让你「不冷场地开口」。

## 💡 五种接话策略

### 1. Paraphrase — 复述确认

**何时用**：对方讲了一大段，你没完全跟上，但又不能干瞪眼。复述一遍 → 既争取思考时间，又确认理解。

**句式模板**：
- *Let me paraphrase to make sure I follow — [你听到的要点].*
- *So, just to play that back — [复述].*
- *In other words, you're saying [复述]?*

> **例**：Let me paraphrase to make sure I follow — we ship the auth refactor first, then the dashboard?

### 2. Build on — 在对方基础上补充

**何时用**：你同意对方，且有相关点要加。这是最「无害」的接话方式——先肯定，再展开，显你在听。

**句式模板**：
- *Building on what [名字] said, [补充点].*
- *To add to that — [补充点].*
- *That's a great point, and I'd take it one step further — [补充点].*

> **例**：Building on what Alex said, the frontend team can mock the API in parallel.

### 3. Clarify — 澄清提问

**何时用**：对方讲的东西有歧义或遗漏。提问是最稳的发言方式——既自然接话，又显你在认真听。

**句式模板**：
- *Quick clarification — [疑问点]?*
- *Could you clarify what you mean by [模糊点]?*
- *Just to be clear — is [复述理解] correct?*

> **例**：Quick clarification — do we own the data migration, or does the platform team handle it?

### 4. Pivot — 承接转向

**何时用**：当前话题快结束，你想把会议引到自己关心的事。关键是先承认对方的话题，再转——显得礼貌不抢话。

**句式模板**：
- *If I could pivot for a second — [新话题].*
- *That ties into something I wanted to raise — [新话题].*
- *Staying on that theme — how does it affect [相关话题]?*

> **例**：If I could pivot for a second — how does this affect the mobile release?

### 5. Hand off — 总结交接

**何时用**：你讲完了，或会议该收尾了。一句总结 + 把话筒交出去，是最专业的收场。

**句式模板**：
- *[总结], so I'll hand off to [名字] for the next part.*
- *To wrap up — [总结]. Over to you, [名字].*
- *That covers my part. [名字], do you want to take it from here?*

> **例**：Mobile stays on the current API until Q4. I'll hand off to Maria now — she owns the mobile roadmap.
