# 读书笔记

> 工程师不只是写代码的人。这里记录我读过的好书——它们塑造思维方式，而思维方式决定代码之外的一切。

---

## 为什么开这个栏目

写代码是手艺，读书是磨刀。技术文章练「怎么干」，读书笔记练「为什么干、怎么活」。

一个好工程师的护城河，往往不在某门语言或框架，而在底层心智——**怎么管理注意力、怎么看待时间、怎么和不确定性共处**。这些东西，书里讲得比文档深。

本栏目不追求面面俱到的书摘搬运，而是**提炼 + 思考 + 行动**：每本书拆出对我最有启发的部分，结合工程师视角写下个人感悟，最后落到可执行的改变。

## 阅读原则

- **不为读完而读**：一本书哪怕只改变一个行为，就值回书价。
- **不复述，要内化**：用自己的话重述，才是真懂。
- **要行动**：读完不动手，等于没读。
