# 技术架构

## 四种技术架构

### 1. 纯 Prompt

最轻量级的方式，直接使用大模型的对话能力。

![](/prompt_arch.png)

### 2. Agent + Function Calling

让大模型具备调用外部工具的能力，扩展其行为边界。

![](/func_arch.png)

### 3. RAG (检索增强生成)

结合外部知识库，让大模型基于检索到的信息回答问题。

![](/embeddings_arch.png)

### 4. Fine-tuning

针对特定任务微调模型参数，获得更精准的表现。

![](/tech_arch.png)
